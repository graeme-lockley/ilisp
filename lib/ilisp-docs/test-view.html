header-content(header-content content)

Extracts the header text from content. Should the content not contain a blank line then the signal InvalidHTTPHeaderContent is raised.

content
string?

The entire content received via an HTTP request.

string?

The header content excluding the blank line.

InvalidHTTPHeaderContent

The content contained no blank line. The first blank signals the end of the header and the start of the body.

find-route(find-round routes request-line)

Locates the first route within routes based on the path captured in request-line and returns that route with the parameters extracted out of path and query string.

If no route matches against the request-line the this procedure returned #f.
